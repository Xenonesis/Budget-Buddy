## âš™ï¸ Installation & Setup

### ğŸ“¦ Detailed Installation Guide

#### Step 1: Clone the Repository

```bash
# Using HTTPS
git clone https://github.com/Xenonesis/Budget-Tracker-.git

# Or using SSH
git clone git@github.com:Xenonesis/Budget-Tracker-.git

# Navigate to the project directory
cd Budget-Tracker-
```

#### Step 2: Install Dependencies

```bash
# Using npm
npm install

# Or using yarn
yarn install

# Or using pnpm
pnpm install
```

#### Step 3: Set Up Supabase

1. **Create a Supabase Account**
   - Visit [supabase.com](https://supabase.com)
   - Sign up for a free account
   - Create a new project

2. **Get Your Credentials**
   - Go to Project Settings > API
   - Copy the Project URL
   - Copy the anon/public key

3. **Run Database Migrations**

   ```bash
   # Option 1: Using Supabase CLI (recommended)
   npm install -g supabase
   supabase login
   supabase link --project-ref your-project-ref
   supabase db push

   # Option 2: Manual SQL execution
   # Run the SQL files in the sql/ directory in order:
   # 1. sql/setup-1-base.sql
   # 2. sql/setup-2-security.sql
   # 3. sql/setup-3-functions.sql
   # 4. sql/setup-ai-tables.sql
   ```

#### Step 4: Configure Environment Variables

Create a .env.local file in the root directory:

```bash
# Copy the example file
cp .env.example .env.local

# Edit the file with your credentials
nano .env.local  # or use your preferred editor
```

**Required Variables:**

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
```

**Optional AI Provider Keys:**

| **notifications**    | User alerts            | Budget, bills, insights         |
| **categories**       | Custom categories      | Icons, colors, hierarchy        |

### ğŸ” Security Features

#### Row Level Security (RLS)

All tables use PostgreSQL Row Level Security to ensure users can only access their own data:

```sql
-- Example RLS Policy
CREATE POLICY "Users can view own data"
  ON transactions FOR SELECT
  USING (auth.uid() = user_id);
```

#### Encryption

- **At Rest**: AES-256 encryption for all data
- **In Transit**: TLS 1.3 for all connections
- **Sensitive Fields**: Additional encryption for financial data

### ğŸ“ˆ Performance Optimizations

#### Indexes

```sql
-- Transaction indexes for fast queries
CREATE INDEX idx_transactions_user_date ON transactions(user_id, date DESC);
CREATE INDEX idx_transactions_category ON transactions(category);
CREATE INDEX idx_transactions_amount ON transactions(amount);

-- Full-text search
CREATE INDEX idx_transactions_search ON transactions
  USING GIN(to_tsvector('english', description));
```

#### Query Optimization

- Composite indexes for common query patterns
- Partial indexes for frequently filtered data
- GIN indexes for array and JSON fields
- B-tree indexes for sorting and range queries

## ğŸ“– Glossary

### Financial Terms

**Assets** Resources owned that have economic value (cash, investments, property)

**Liabilities** Financial obligations or debts owed to others

**Net Worth** Total assets minus total liabilities

**Cash Flow** Movement of money in and out of your accounts

**Budget** A financial plan outlining expected income and expenses

**Emergency Fund** Savings set aside for unexpected expenses (3-6 months of expenses)

**Compound Interest** Interest earned on both principal and accumulated interest

**Diversification** Spreading investments across different assets to reduce risk

**Liquidity** How quickly an asset can be converted to cash

**ROI (Return on Investment)** Percentage gain or loss on an investment

**Expense Ratio** Annual fee charged by investment funds (as percentage of assets)

**Portfolio** Collection of financial investments

**Bull Market** Period of rising prices in financial markets

**Bear Market** Period of falling prices in financial markets

**Inflation** Rate at which prices increase over time

**Deflation** Rate at which prices decrease over time

**APR (Annual Percentage Rate)** Yearly interest rate including fees

**APY (Annual Percentage Yield)** Actual annual return including compound interest

**Credit Score** Numerical representation of creditworthiness (300-850)

**Debt-to-Income Ratio** Monthly debt payments divided by gross monthly income

### Budget Buddy Terms

**Category** Group of similar expenses (e.g., Food & Dining, Transportation)

**Subcategory** Specific type within a category (e.g., Restaurants, Coffee)

**Transaction** Record of money spent or received

**Recurring Transaction** Expense or income that happens regularly (subscriptions, salary)

**Budget Period** Time frame for a budget (daily, weekly, monthly, yearly)

**Rollover** Carrying unused budget amount to the next period

**Alert Threshold** Percentage of budget that triggers a notification

**Envelope Budgeting** Allocating specific amounts to different spending categories

**Zero-Based Budget** Budget where income minus expenses equals zero

**Goal** Financial target you're working toward

**Milestone** Progress marker on the way to a goal

**AI Insight** Recommendation or observation from artificial intelligence

```typescript
function calculateRetirement(params) {
  const {
    currentAge,
    retirementAge,
    currentSavings,
    monthlyContribution,
    annualReturn,
    inflationRate,
    desiredMonthlyIncome,
    yearsInRetirement,
  } = params;

  const yearsToRetirement = retirementAge - currentAge;
  const realReturn = annualReturn - inflationRate;

  // Future value of current savings
  const futureValueCurrent = currentSavings * Math.pow(1 + realReturn, yearsToRetirement);

  // Future value of monthly contributions
  const monthlyRate = realReturn / 12;
  const months = yearsToRetirement * 12;
  const futureValueContributions =
    monthlyContribution * ((Math.pow(1 + monthlyRate, months) - 1) / monthlyRate);

  const totalSavings = futureValueCurrent + futureValueContributions;

  // Required savings for desired income
  const requiredSavings = (desiredMonthlyIncome * 12 * yearsInRetirement) / (1 + realReturn);

  return {
    projectedSavings: totalSavings,
    requiredSavings: requiredSavings,
    surplus: totalSavings - requiredSavings,
    onTrack: totalSavings >= requiredSavings,
    monthlyIncomeProjected: totalSavings / yearsInRetirement / 12,
  };
}
```

```typescript
function calculateMortgageAffordability(params) {
  const {
    annualIncome,
    monthlyDebts,
    downPayment,
    interestRate,
    loanTerm, // years
    propertyTax,
    insurance,
    hoa,
  } = params;

  // 28/36 Rule
  const maxHousingPayment = monthlyIncome * 0.28;
  const maxTotalDebt = monthlyIncome * 0.36;
  const maxMortgagePayment =
    Math.min(maxHousingPayment, maxTotalDebt - monthlyDebts) - propertyTax - insurance - hoa;

  // Calculate max loan amount
  const monthlyRate = interestRate / 12;
  const payments = loanTerm * 12;
  const maxLoanAmount =
    maxMortgagePayment * ((1 - Math.pow(1 + monthlyRate, -payments)) / monthlyRate);

  const maxHomePrice = maxLoanAmount + downPayment;

    maxHomePrice: maxHomePrice,
    maxLoanAmount: maxLoanAmount,
    downPayment: downPayment,
    monthlyPayment: maxMortgagePayment,
    monthlyTotal: maxMortgagePayment + propertyTax + insurance + hoa,
    debtToIncomeRatio: (maxTotalDebt / monthlyIncome) * 100,
    recommended: maxHomePrice * 0.8, // Conservative recommendation
    aggressive: maxHomePrice * 1.1, // Aggressive scenario
  };
}
```

```typescript
function calculateTaxOptimization(params) {
  const {
    income,
    filingStatus,
    traditionalIRA,
    rothIRA,
    contributions401k,
    hsa,
    charitableDonations,
    mortgageInterest,
    stateTaxes,
  } = params;

  // Calculate AGI
  const adjustments = traditionalIRA + contributions401k + hsa;
  const agi = income - adjustments;

  // Standard vs Itemized Deduction
  const standardDeduction = filingStatus === 'married' ? 27700 : 13850;
  const itemizedDeduction = charitableDonations + mortgageInterest + stateTaxes;
  const deduction = Math.max(standardDeduction, itemizedDeduction);

  const taxableIncome = Math.max(0, agi - deduction);

  // Calculate tax (simplified)
  let federalTax = 0;
  const brackets = [
    { limit: 11000, rate: 0.1 },
    { limit: 44725, rate: 0.12 },
    { limit: 95375, rate: 0.22 },
    { limit: 182100, rate: 0.24 },
    { limit: 231250, rate: 0.32 },
    { limit: 578125, rate: 0.35 },
    { limit: Infinity, rate: 0.37 },
  ];

  let remaining = taxableIncome;
  let previousLimit = 0;

  for (const bracket of brackets) {
    const taxableInBracket = Math.min(remaining, bracket.limit - previousLimit);
    federalTax += taxableInBracket * bracket.rate;
    remaining -= taxableInBracket;
    previousLimit = bracket.limit;
    if (remaining <= 0) break;
  }

  return {
    agi: agi,
    taxableIncome: taxableIncome,
    federalTax: federalTax,
    effectiveRate: (federalTax / income) * 100,
    taxSavings: {
      from401k: contributions401k * 0.22, // Estimated marginal rate
      fromHSA: hsa * 0.22,
      fromIRA: traditionalIRA * 0.22,
      total: (contributions401k + hsa + traditionalIRA) * 0.22,
    },
    recommendations: [
      contributions401k < 23000 ? 'Max out 401(k)' : null,
      hsa < 4150 ? 'Max out HSA' : null,
      traditionalIRA < 7000 ? 'Consider IRA contribution' : null,
    ].filter(Boolean),
  };
}
```

```
Results: Transactions containing any of these terms
```

#### 4. Multiple Terms (AND)

**Using Plus:**

```
Search: "restaurant + dinner"
Results: Transactions containing both terms
```

#### 5. Exclude Terms

**Using Minus:**

```
Search: "food -restaurant"
Results: Food transactions excluding restaurants
```

#### 6. Amount Range

**Using Operators:**

```
Search: amount:>100
Results: Transactions over $100

Search: amount:50-150
Results: Transactions between $50 and $150

Results: Transactions $25 or less
```

**Filter by Category:**

```
Search: category:"Food & Dining"

```
